Module axi_lite_xbar

axi_lite_xbar: Fully-connected AXI4-Lite crossbar. See doc/axi_lite_xbar.md for the documentation, including the definition of parameters and ports.

Parameters

NumSlvPorts: int unsigned

Number of slave ports of axi_lite_xbar. This many master modules are connected to it.

NumMstPorts: int unsigned

Number of master ports of axi_lite_xbar. This many slave modules are connected to it.

MaxMstTrans: int unsigned

The max number of open transactions each master connected to a slave port can have in flight.

MaxSlvTrans: int unsigned

The max number of open transactions each master port can generate to a slave module connected.

FallThrough: bit

The internal FIFOs are in fallthrough mode.

LatencyMode: axi_pkg::xbar_latency_e

The Latency mode of the xbar. This determines if the channels on the ports have a spill register instantiated. Example configurations are provided with the enum axi_pkg::xbar_latency_e.

AxiAddrWidth: int unsigned

AXI4-Lite address field width.

AxiDataWidth: int unsigned

AXI4-Lite data field width.

NumAddrRules: int unsigned

The number of address rules in the address map of the xbar. Each master port should have at least one rule specified.

aw_chan_t: type

AXI4-Lite AW channel type.

w_chan_t: type

AXI4-Lite W channel type.

b_chan_t: type

AXI4-Lite B channel type.

ar_chan_t: type

AXI4-Lite AR channel type.

r_chan_t: type

AXI4-Lite R channel type.

req_t: type

AXI4-Lite request struct type.

resp_t: type

AXI4-Lite response struct type.

rule_t: type

Address decoding rule.

DefaultIdxWidth: int unsigned

DEPENDENT PARAMETERS, DO NOT OVERWRITE! Default master port index width.

default_idx_t: type

DEPENDENT PARAMETERS, DO NOT OVERWRITE! Index type of the default master port index.

Ports

clk_i: input logic

Clock, positive edge triggered.

rst_ni: input logic

Asynchronous reset, active low.

test_i: input logic

Testmode enable, active high.

slv_ports_req_i: input req_t [NumSlvPorts-1:0]

AXI4-Lite slave ports, requests.

slv_ports_resp_o: output resp_t [NumSlvPorts-1:0]

AXI4-Lite slave ports, responses.

mst_ports_req_o: output req_t [NumMstPorts-1:0]

AXI4-Lite master ports, requests.

mst_ports_resp_i: input resp_t [NumMstPorts-1:0]

AXI4-Lite master ports, responses.

addr_map_i: input rule_t [NumAddrRules-1:0]

Address map array input for the crossbar. This map is global for the whole module. It is used for routing the transactions to the respective master ports. Each master port can have multiple different rules.

en_default_mst_port_i: input logic [NumSlvPorts-1:0]

Enables a default master port for each slave port. When this is enabled unmapped transactions get issued at the master port given by default_mst_port_i.

When not used, tie to '0.

default_mst_port_i: input default_idx_t [NumSlvPorts-1:0]

For each slave port the default index where the transaction should be routed, if for this slave port the default index functionality is enabled by setting the bit en_default_mst_port_i[slave_port_idx] to '1.

When not used, tie to '0.
