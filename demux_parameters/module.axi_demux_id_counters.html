Module axi_demux_id_counters

Internal ID counter module of axi_demux.

There needs to be kept track of which IDs are currently in-flight at a given master port of the module so that the AXI ordering requirements are not violated.

Parameters

AxiIdBits: int unsigned

The lower bits of the AXI ID that should be considered, results in 2**AXI_ID_BITS counters.

CounterWidth: int unsigned

The width of each instantiated counter. This equates to the maximum number of transactions which can be in flight of a given ID.

mst_port_select_t: type

The type of the selection signal to indicate on which master port the ID is transmitted.

NumCounters: int unsigned

Ports

clk_i: input logic

Clock, rising edge triggered.

All other signals (except rst_ni) are synchronous to this signal.

rst_ni: input logic

Asynchronous reset, active low.

lookup_axi_id_i: input logic [AxiIdBits-1:0]

The new ID that should be looked up for if it is already in-flight.

lookup_mst_select_o: output mst_port_select_t

The master port where the looked up transaction is currently in-flight if lookup_mst_select_occupied_o is 1.

lookup_mst_select_occupied_o: output logic

When 1, the looked up ID is currently in-flight on master port indicated by lookup_mst_select_o.

full_o: output logic

The counter selected by push_axi_id_i is currently full.

push_axi_id_i: input logic [AxiIdBits-1:0]

The ID that selects the counter.

push_mst_select_i: input mst_port_select_t

Specify the master port where the ID is transmitted.

push_i: input logic

Increment the counter.

inject_axi_id_i: input logic [AxiIdBits-1:0]

Inject the ATOP id in AR channel. This port should only be used on the AR channel, tie to 0 in AW channel.

inject_i: input logic

Push the ID, this increments the counter selected by inject_axi_id_i by an additional +1.

pop_axi_id_i: input logic [AxiIdBits-1:0]

The ID of the counter that should be decremented on transaction completion.

pop_i: input logic

Decrement the counter selected with pop_axi_id_i.
